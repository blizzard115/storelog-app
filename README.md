# 共有ノート

## 📝 概要
共有ノートは、アパレル店舗における情報共有を効率化するためのWebアプリケーションです。

紙の連絡ノートをデジタル化し、
「誰が読んだか」「誰がまだ読んでいないか」を可視化することで、
伝達漏れや確認漏れを防ぎます。

さらに、店舗単位でデータを分離することで、
他店舗の情報が混ざらない安全な共有環境を実現しています。

---

## 🎯 作成背景
アパレル店舗での勤務経験の中で、以下の課題を感じました。

- 情報共有が紙のノートで管理されている
- 読んだかどうかの確認が手書きで曖昧
- 「読んだけどサインしない」問題が発生する
- ノートを遡るのが大変
- LINEでの共有は休みの日にも通知が来てしまう
- ミス共有が個人攻撃のように感じられるケースがある

これらの課題を解決するために、
「既読管理」「未読の可視化」「店舗単位の情報管理」を備えた
共有ノートを開発しました。

---

## ✨ アプリの特徴

- 🏬 店舗単位でデータを分離（他店舗の情報は見えない）
- 👀 既読管理により「誰が確認したか」が分かる
- 🚨 未読ユーザーの可視化で確認漏れを防止
- 📱 シンプルなUIで現場でも使いやすい設計
- 🔍 紙ノートの課題（検索性・修正性・心理的負担）を改善

---

## ⚙️ 主な機能

### 🏬 店舗管理機能
- 店舗の新規作成
- 店舗コードによる参加機能
- 店舗ごとに投稿・ユーザーを分離

### 📢 投稿機能（連絡・共有）
- 店舗内の情報を投稿（連絡事項・入荷情報・クレーム対応・接客メモ）
- タイトル・本文で簡単に共有
- 投稿は店舗単位で共有

### 👀 既読機能
- 投稿ごとに「確認しました」ボタン
- 誰が確認したかを可視化

### 🚨 未読管理
- 未読ユーザーの一覧表示
- 未読投稿のみの絞り込み表示
- 確認漏れを防止

### 🔍 検索・フィルタ機能
- キーワード検索
- 投稿タイプ別の絞り込み
- 未読のみ表示

### 👤 ユーザー管理
- ユーザーごとの投稿・既読管理

- Docker

---

### 💡 工夫した点
- 店舗単位でデータを分離し、実運用を意識した設計にした
- 既読・未読を可視化し、紙ノートの「サイン問題」を解決
- UIをシンプルにし、現場スタッフでも直感的に使える設計にした
- N+1問題を防ぐために includes を使用
- UUIDではなく用途に応じた設計を選択し、可読性を重視
